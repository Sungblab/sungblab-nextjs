# 학교 커뮤니티 웹사이트를 만들게 된 계기

안녕하세요, Sungblab입니다! 완도고등학교 학생들을 위한 커뮤니티 사이트 '완대전' 개발 이야기를 공유하려고 합니다. 기존의 여러 커뮤니티 사이트들이 있었지만, 우리 학교만의 특화된 기능들이 필요하다고 생각했습니다.

#### 기술 스택 선택

프로젝트를 시작하면서 가장 중요하게 생각한 것은 구조가 쉽고, 빠르게 만들어 보고자 했습니다. 이를 위해 다음과 같은 기술 스택을 선택했습니다:

- **Frontend**: Vanilla HTML
- **Backend**: Express.js & Node.js
- **Database**: MongoDB
- **인증/보안**: JWT & bcrypt
- **UI/스타일링**: Tailwind CSS

## 제작 과정

### 1. 기획

저는 친구랑 이야기를 나누던중, 자기 학교의 자기 학교만의 커뮤니티와 다양한 시스템이 존재하단걸 우연히 듣게 되었습니다.
과구리 라는 서비스더군요. 저는 그래서 저도 나만의 내학교 커뮤니티 사이트를 만들어 보고 싶다고 생각했습니다.

![image.png](/images/과구리.png)

이게 제가 벤치마킹 한 사이트 입니다. 저는 이걸 보고 제가 만들고 싶은 커뮤니티 사이트의 모습을 생각하게 되었습니다.

그래서 전 여름방학에 개발에 착수했습니다. 그때가 claude 3.5 sonnet 이 처음 나왔을때였습니다. 그래서 이걸 활용해서
계속 만들어 나갔습니다. 초기 개발을 완료하고 테스트를 해보고, 프론트는 네트리파이에 백엔드는 클라우드타입에 호스팅을
진행해 보았습니다. 결과는 성공적 이였습니다. 물론 커뮤니티 기능만 빼고요 이부분은 더 개발해보기로 하고 제이름 걸고
학교에 포스터 걸고 홍보도 해보았습니다. (물론 아무도 쓰진 않습니다..)

### 2. 개발

![image.png](/images/완대전.png)

이건 메인페이지 입니다. 레이아웃이 거의 비슷하죠? ㅎㅎ 초안 제작및 테스트는 클로드와 함께 하니 일주일만에
완성을 해보았습니다. 제 첫 풀스택 프로젝트인데요 나름 만족스럽게 만든거 같습니다. 저희 학교의 상징색인 푸른색
을 테마로 적용시켰습니다. 다른걸 설명하기 이전에 급식 부분은 나이스 OPEN api 에서 키를 발급받아 비교적
쉽게 사용할수 있었습니다. 백엔드에서 api 짜서 호출했습니다.

![image.png](/images/공지사항.png)
관리자 계정으로 로그인한 상태입니다. 공지사항 작성 페이지구요 목록에서 작성및 수정이 가능합니다.

![image.png](/images/청원.png)
청원 페이지구요 여기선 말 그대로 청원을 요청하고 사람들의 같이 청원을 투표할수 있습니다. 여긴 로그인 한
사람만 접속 가능합니다.

![image.png](/images/도우미.png)
여긴 학급 도우미 페이지 인데요 대포뽑기, 타이머, 무작위 팀 뽑기 기능등 여러가지 간단한 기능들을 모아놓은 페이지 입니다.

![image.png](/images/소개.png)
여긴 소개 페이지 입니다. 네 사진에서 보다싶이 이런것들과 맨 아래에는 개발자 연혁이 있습니다.

![image.png](/images/커뮤.png)
네 여긴 대망의 커뮤니티 페이지 인데요 제가 이건 고민을 많이 했습니다. 익명 게시판으로는 만들고 싶은데 그러면 또
문제가 생길거같고, 로그인을 하고 익명으로 글쓰면 글쓴이가 누군지 아니 운영자인 나를 믿지 못할것이고, 또 그렇다고
로그인 없이 글을 쓸수 있게하면 난장판이 될거같고 고민이 많았지만 그러면 차라리 DC인사이드와 같이 IP기반 익명 시스템을
도입 하자는 생각이 떠올랐습니다. 그래서 구현을 시도하고 몇일간 머리를 싸맨후에 결국 완성 했습니다.

![image.png](/images/글쓰기.png)
이미지 업로드 기능과 함께 글을 쓸수 있도록 만들어 놨고요, 로그인을 했으면 무조건 실명
비로그인 자는 익명으로 글을 쓸수 있게 설정 해놓았습니다. DC인사이드 시스템을 참고했어요

![image.png](/images/글.png)
여기도 DC인사이드 시스템을 참고해보았는데요 추천 시스템, 댓글도 비슷하게 만들었습니다.
참고로 주간 베스트 시스템도 있습니다.

### 2. 마무리

네 제가 이건 학교 동아리 활동의 일환으로 제작 해보았는데 아직은 활성화가 되진 않았지만 성공적으로 만들고 운영하고 있어서
기분은 좋습니다. 아직 서비스를 운영중이라 만약 코드가 궁금하시다면 개인적으로 연락을 취해주시면 감사하겠습니다.
